# Breakthrough — Deployment Package

This folder is the **public deployment package** for the Breakthrough demonstration — deliberately separated from the rest of the Obsidian vault (research notes, decision logs, assumptions) so that only demo-appropriate material is ever pushed to a public GitHub repository.

## Structure

```text
Deployment/
├── streamlit_app/        # The interactive demo — deploy to Streamlit Community Cloud
│   ├── app.py             # Main Streamlit application
│   ├── requirements.txt   # Python dependencies
│   ├── pipeline/           # Copy of the reconciliation pipeline (see Working Files/pipeline in the vault)
│   └── data/                # Bundled synthetic source files (both demo months)
├── landing_page/          # The public-facing static site — deploy to Vercel
│   ├── index.html
│   └── vercel.json
└── README.md              # This file
```

## What This Is

A fully synthetic, working demonstration of the Breakthrough back-office reporting pipeline for "Ferrous Home Services," a fictional home-service contractor. No real business, customer, vendor, or employee data appears anywhere in this folder.

## Before You Deploy

1. Both `streamlit_app/` and `landing_page/` have been tested **locally** in the build sandbox (see `Deployment Guide.md` in the main vault for the full test record). They have not yet been deployed to a live, public URL.
2. `landing_page/index.html` contains a placeholder, `STREAMLIT_APP_URL_PLACEHOLDER`, in two places (the two "Open the Live Demo" buttons). **Replace this with your real Streamlit Community Cloud URL after deploying the app**, before deploying the landing page.
3. See `Deployment Guide.md` (in the vault root) for exact, step-by-step instructions to push this to GitHub and deploy both pieces for free.

## Reproducing the Demo Data

The bundled synthetic data in `streamlit_app/data/` was generated by `Working Files/scripts/generate_synthetic_data.py` in the main vault (seeded, reproducible — seed=42). Do not replace it with real business data; this deployment is for demonstration purposes only.
